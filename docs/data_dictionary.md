# Data Dictionary

## Approved Columns for Analysis

### Transaction_ID
Unique identifier for each transaction.

### Customer_ID
Unique identifier for customer.

### Age
Customer's age in years.

### Gender
Customer's gender (demographic information).

### Income
Customer's annual income level.

### Customer_Segment
Customer segmentation category (e.g., Premium, Standard, Basic).

### Date
Transaction date in full format.

### Year
Year when the transaction occurred.

### Month
Month when the transaction occurred.

### Time
Time when the transaction occurred.

### Product_Category
Category of product purchased (e.g., Electronics, Clothing, Home & Garden).

### Brand
Brand name of the purchased product.

### Type
Product type or subcategory within the main category.

### Amount
Individual product amount before any calculations.

### Total_Amount
Total revenue generated from the transaction.

### Shipping_Method
Method used for product delivery (e.g., Standard, Express, Overnight).

### Payment_Method
Payment method used for the transaction (e.g., Credit Card, Debit Card, Cash).

### Order_Status
Current status of the order (e.g., Delivered, Pending, Cancelled).

### Ratings
Customer rating or satisfaction score for the transaction/product.

## PII Columns (Not Used in Analysis)

### Name
Customer's full name (excluded for privacy).

### Email
Customer's email address (excluded for privacy).

### Phone
Customer's phone number (excluded for privacy).

### Address
Customer's physical address (excluded for privacy).

### Zipcode
Customer's postal/zip code (excluded for privacy).
